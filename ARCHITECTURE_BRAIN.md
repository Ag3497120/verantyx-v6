# Verantyx Brain Architecture — 170K行フル活用設計

## 思想

V3は「手」、verantyxは「目と脳」。
V3にコードを書かせるが、**何を書くべきかはverantyxが決める**。

## パイプライン

```
Task → [Phase A: verantyx全力分析] → [Phase B: 戦略決定] → [Phase C: V3に指示] → [Phase D: 検証+修正]
```

---

## Phase A: verantyx全力分析（170K行の真価）

### A1. Cross Engine トライアル（94モジュール）
- 全94モジュール実行、各モジュールの**中間結果**を収集:
  - pieces生成数、部分一致スコア（train[0]との一致率）
  - 最も近い結果のグリッド差分
- 「neighborhood_ruleが60%一致」「flood_fillがサイズだけ合った」等

### A2. Cross2 分解（8種 × 全train例）
- obj4, obj8, mono, pan_h, pan_v, colors, enclosed, whole
- 入力と出力**両方を分解**して構造対応を取る:
  - 入力3obj → 出力3obj（1:1対応？）
  - オブジェクトの移動・回転・色変更を検出

### A3. World Commands スキャン（216コマンド）
- 単体: 216 × train[0]
- 2段合成: top-20 × 216 = 4320通り
- 収束: 繰り返し適用
- **閾値30%以上**の部分一致を全て報告

### A4. World Laws / Priors（43法則）
- world_priors.pyの43ピースを全て試行

### A5. Cross Solver DSLパターン
- Phase 1のcross_solverがどのパターンに反応したか
- 生成されたDSL式（部分的でも）

### A6. CEGIS / ProgramTree 探索
- Phase 8-9の条件分岐合成を短時間(5秒)実行

### A7. 600B SVD 推論型分類
- Expert活性化パターンから問題の「認知的カテゴリ」を推定

### A8. 構造的特徴抽出
- セパレータ、周期性、対称性、入出力構造対応

## Phase B: 戦略決定（cross構造によるルーティング）

分析結果をcross構造ピースとして組み立て、戦略テンプレートを選択:
1. **修正戦略**: 既存ソルバー70%+一致 → 差分修正を指示
2. **合成戦略**: obj 1:1対応あり → 各objの変換を指示
3. **パネル戦略**: パネル分解成功 → パネル間関係を指示
4. **DSL拡張戦略**: DSLが部分的に動いた → 拡張を指示
5. **フルスクラッチ**: 何も引っかからない → 基本分析のみ

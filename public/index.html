Verantyx-CLI in detail

A local-first AI harness (Omni CLI): routing, deliberation, tools, and cross-session memory — built for machines that cannot keep a frontier model loaded all day.

What it is

  • Resident small router; larger local models on demand
  • Classify-only intent routing (the router does not invent answers)
  • Vector council / optional puzzle axes / agent tools
  • Eternal memory + feedback & reviewer loops
  • Published honest benchmarks and claim boundaries

What it is not

  • Not an accuracy booster that turns 0.5B into a frontier model
  • Not “structure equals more world knowledge”
  • Not a one-click cloud demo — real local setup, real tradeoffs

Prefer the curated stable branch or v3.0.0-alpha. main moves fast as a research workbench.

💡 What Makes Verantyx-CLI Different?

Traditional AI assistants forget your conversations. Verantyx-CLI remembers everything using a revolutionary Cross Structure that organizes knowledge across six dimensions:

  • UP — User inputs (what you asked)
  • DOWN — AI responses (what it answered)
  • LEFT — Temporal data (when conversations happened)
  • RIGHT — Action records (tools used, files modified)
  • FRONT — Active conversation threads
  • BACK — Historical context and metadata

Every interaction is stored in .jcross files — a unified language where code, data, and logic coexist. No separate databases. No JSON bloat. Just pure, human-readable Cross Structure.

🇯🇵 Built by a Student in Kyoto

Verantyx is built by kofdai, a student in Kyoto, Japan, who doesn't have access to massive GPU clusters or enterprise API budgets. Instead, he uses Claude as a development partner — designing the architecture himself, then collaborating with AI to implement, test, and iterate at unprecedented speed.

One laptop. One human vision. One AI pair-programmer. Competing at the frontier of AGI research.

The engine itself is 100% symbolic — no LLM is used in the solving pipeline. AI assists only in the development process. This is what human-AI collaboration looks like when the human brings the architectural insight and the AI brings implementation velocity.
